# 日用品在庫管理アプリ - 要件定義書

## 1. 概要

### 目的
日用品（ティッシュ、洗剤など）のストック状況をWebアプリで管理し、スマートフォンから手軽に確認・更新できるようにする

### 対象ユーザー
- 初期リリース: 個人利用
- 将来: 家族・ルームメイトとの共有

---

## 2. 機能要件

### 2.1 MVP（初期リリース）

| 機能 | 説明 |
|------|------|
| 認証 | メール+パスワードでのログイン/サインアップ |
| 品目一覧 | カード形式で在庫状況を表示 |
| 品目登録 | 新規品目の追加 |
| 品目編集 | 既存品目の情報変更 |
| 品目削除 | 不要な品目の削除 |
| 在庫増減 | +1/-1ボタンでワンタップ操作 |
| 並び替え | ドラッグ&ドロップで品目の順番を変更 |

### 2.2 品目の登録項目

| 項目 | 必須 | 型 | 説明 |
|------|------|------|------|
| 品目名 | ○ | string | 例: シャンプー 詰め替え |
| 商品名 | - | string | 例: YOLU カームナイトリペア |
| 価格 | - | number | 購入価格（円） |
| 在庫数 | ○ | number | 現在の在庫数 |
| 単位 | ○ | string | 個、本、箱、パックなど |
| 備考 | - | string | 自由記述 |

### 2.3 将来追加予定

| 機能 | 優先度 |
|------|--------|
| 複数ユーザー共有 | 高 |
| 在庫少通知 | 高 |
| 購入履歴 | 中 |
| カテゴリ分類 | 中 |
| 商品画像 | 低 |
| 購入リンク | 低 |

---

## 3. 非機能要件

| 項目 | 要件 |
|------|------|
| 対応デバイス | スマートフォン優先、PC/タブレットも対応 |
| セキュリティ | 認証必須、HTTPS通信 |
| 可用性 | Vercelの標準SLA |

---

## 4. 技術スタック

| カテゴリ | 技術 |
|----------|------|
| フレームワーク | Next.js 15 (App Router) |
| 言語 | TypeScript |
| ORM | Prisma 7 |
| DB | PostgreSQL (Supabase) |
| 認証 | Better Auth |
| UI | shadcn/ui + Tailwind CSS |
| ホスティング | Vercel |

### 実装方針
- Server Actionsを積極活用、API Routesは最小限
- Server Componentを優先

---

## 5. スコープ外

- ロット数量の管理
- Excelからのデータインポート
- 複数ユーザー共有（初期リリース時）
- 通知機能（初期リリース時）
